# Privacy model

job-hunter handles some of the most sensitive data a person owns: their complete career history and
contact details. The whole product is built so that data **stays with the user**.

## Principles

1. **Local-first.** The application is a desktop app. The master CV, imported jobs, generated
   documents, and the database all live on the user's device. There is no backend that receives
   personal data.
2. **Encrypted at rest.** Local storage uses SQLite + SQLCipher.
3. **No PII in this repository — ever.** This is a public repo. It is developed and tested entirely
   against **synthetic** data generated by `tools/fake-data`. A `.gitignore` PII firewall blocks real
   CV files, secrets, local databases, and personal assets from being committed.
4. **Compliant acquisition.** Jobs are brought in by user-driven capture (browser extension clip,
   paste, saved-search email) — never by automated login or scraping. No third-party credentials are
   stored or required.
5. **AI is optional and bounded.** The Applicant Advocate AI layer is feature-flagged and off by
   default. It runs against a **local model (Ollama)** or a **user-supplied API key**. When a cloud
   model is used, only the minimal redacted evidence needed for the task is sent — never the full CV
   or raw contact details — and the user opts in explicitly.

## The PII firewall (`.gitignore`)

The repository's `.gitignore` blocks, among others:

- `private/`, `*.real.*`, `*.local.*`, `master-cv.json`, `cv.json`, `resume.*`, `my-cv.*`
- secrets: `.env*`, `*.key`, `*.pem`, `secrets/`
- local databases: `*.sqlite*`, `*.db`
- personal assets: `*.headshot.*`, `profile-photo.*`, `private-assets/`

Only files under `fixtures/` (explicitly synthetic) may contain CV-shaped data.

## For contributors

- **Never commit real CV data, real names, real contact details, or screenshots containing them.**
- Use `node tools/fake-data/generate.js` to produce test data.
- A PII/secret sweep should pass before any commit (e.g. SENTINEL's `pii-audit` / `secret-scan`, or a
  grep for obvious PII patterns).

## For users

Your real master CV lives outside this repository — in the app's local data directory at runtime, or
wherever you keep your own files. The tool reads it locally to do its work and writes your tailored
documents locally. Nothing is uploaded.
